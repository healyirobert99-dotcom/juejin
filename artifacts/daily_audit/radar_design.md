# 信号雷达设计与实现说明

## 概述

新增"信号雷达 · 接近触发"区块，在正式信号检测逻辑之上，识别当前已满足 2/3 四因子条件的行业，作为观察区。

## 设计原则

- **仅观察、不交易**：雷达结果不影响 signals / live_trades / ETF 匹配等系统
- **与正式策略隔离**：使用 `radar_candidates` 命名，与 `today_signals` 完全隔离
- **数据一致**：使用与正式策略完全相同的 `industry_daily` 数据和 `signal_data_date` 截止
- **低强调度**：视觉上使用橙色/灰色，不占用大红大绿色

## 关键算法

### 硬性资格（全部通过才进入雷达）

1. 行业股票数 >= 20（与正式信号一致）
2. 行业历史日记录 >= 310（warmup_days + lookback_days）
3. 不在 60 日信号冷却期内
4. 过去 60 个行业交易日中宽度 < 35% 的天数 >= 30
5. 前一个行业交易日宽度 < 35%

### 当前三条件（至少满足 2/3）

1. 当前宽度 >= 35%
2. 当前平均量比 >= 1.0
3. 当前行业平均 20 日涨幅 > 0

### 排序规则

1. 条件通过数量降序 → 2. 近5日宽度改善幅度降序 → 3. 当前宽度降序 → 4. 行业名称升序

### 限制

- 最多 5 个行业
- 已产生正式信号的行业不重复显示
- 数据阻断时整个区块不显示

## 测试覆盖

13 条测试全部通过：
1. 正式信号不变
2. 雷达数据截止
3. 行业规模过滤
4. 310 日历史门槛
5. 冷却期过滤
6. 弱势背景检查
7. 三条件至少 2/3
8. 正式信号去重
9. 最多 5 个
10. 排序稳定
11. 空状态
12. 阻断不显示
13. 不生成交易
